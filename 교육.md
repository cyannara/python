LLM 기반 AI 에이전트는 원하는 결과를 달성하기 위해 LLM 호출을 포함한 여러 처리 단계를 연결하는 소프트웨어 시스템입니다. 조건부 로직이나 의사결정 기능을 갖추고 있으며, 단계 간에 접근할 수 있는 작업 메모리(Working Memory)를 보유합니다. 에이전트의 핵심 특성은 자율성입니다. 단일 프롬프트에 반응하고 끝나는 것이 아니라, 환경을 평가하고 새로운 정보가 들어오면 실시간으로 계획을 수정하며, 목표를 달성할 때까지 여러 턴에 걸쳐 자율적으로 일련의 행동을 수행합니다. ### LLM 리터러시

- ML/DL 데이터 흐름 이해
  - (이론) 컴퓨터가 이해하는 데이터: 스칼라, 벡터, 행렬, 텐서(Tensor)의 개념
  - (실습) RunPod 환경에서 NumPy/PyTorch 텐서 생성 및 차원 변환 실습
  - (이론)입력(X)과 가중치(W)의 곱: 데이터가 예측값으로 변하는 가장 단순한 흐름
  - (실습)y = Wx + b를 드로 구현하여 단순 수치 예측 모델 구축
  - (이론)비선형성(ReLU, Sigmoid) 추가:
    데이터 흐름에 '판단'의 기준을 넣는
    이유-(실습)활성화 함수 유무에 따른
    데이터 매핑 그래프 시각화 비교

- Self-Attention 메커니즘 및 디코더 중심 아키텍처 분석
- Decoding 전략 및 프롬프트 제어 기법

### LLM 리터러시

대규모 데이터를 학습한 인공지능과 효과적으로 소통하고, 생성된 결과물을 비판적으로 평가하며, 이를 자신의 목적에 맞게 책임감 있게 활용할 수 있는 종합적인 능력

Literacy(문해력)

`주요 구성 요소`

- 정확한 소통(프롬프트 엔지니어링): LLM이 원하는 맥락과 형식의 답변을 도출할 수 있도록 명확하고 구체적인 지시사항(프롬프트)을 작성하는 능력
- 비판적 사고 및 결과 평가: AI가 생성한 정보에 오류(환각 현상, Hallucination)나 편향이 있을 수 있음을 인지하고, 정보의 사실 여부(팩트체크)를 교차 검증하는 능력
  0 원리 및 한계 이해: LLM이 인간처럼 '생각'하는 것이 아니라 방대한 데이터를 바탕으로 확률적으로 단어를 예측하고 조합한다는 작동 방식을 이해하는 것
- 윤리적 활용: 민감한 개인정보나 기밀 데이터를 AI에 유출하지 않고, 저작권 및 윤리적 가이드라인을 준수하여 책임감 있게 사용하는 능력
